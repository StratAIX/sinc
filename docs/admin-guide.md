## 【初回のみ】アカウント作成手順

### ① 管理者アカウント（admin.html専用）

1. [Supabase Dashboard](https://supabase.com/dashboard) → 対象プロジェクト → `Authentication` → `Users`
2. `Add user` → `Create new user`
   - Email: `info@strataix.net`
   - Password: 強いパスワードを設定
   - **Auto Confirm User: ✅ チェックを入れる**（メール確認スキップ）
3. `Table Editor` → `users` テーブルに手動でレコードを Insert
   ```
   id         : Authentication で作成されたユーザーのUUID（Authページでコピー）
   is_admin   : true
   ban_status : active
   （他の列は空欄でOK。診断スコア不要）
   ```
4. これで `sinc.strataix.net/admin.html` にログインできるようになる

### ② GMアカウント（掲示板の「Sinc公式」用）

1. `sinc.strataix.net` から **普通に診断→登録**（136問を受ける）
2. ニックネームを「Sinc公式」などに設定
3. αテスト中の掲示板の最初の書き込みや場の雰囲気作りに使う
4. このアカウントに管理者権限は**不要**（admin.htmlとは別運用）

> **なぜ分けるか**: 管理アカウントとGMアカウントを分けることで、片方の情報が漏れてももう片方に影響が出ない。管理者権限は最小限のアカウントにのみ付与する。

## ログイン方法

1. ブラウザで `https://sinc.strataix.net/admin.html` にアクセス
2. **未ログインの場合** → 自動で `index.html` にリダイレクトされるので、`info@strataix.net` でログインしてから再アクセス
3. **ログイン済みの場合** → 管理者メールアドレス（`info@strataix.net`）かつ DB の `is_admin = true` の場合のみ表示される

> ⚠️ 管理者を追加したい場合は `admin.html` の `ADMIN_EMAILS` 配列にメールアドレスを追記し、Supabase の `users` テーブルで該当ユーザーの `is_admin` を `true` に設定する

## 各機能の使い方

### 📊 概要（起動時の確認ルーティン）

- **アラートバナー** が表示されていたら通報件数が多い → 通報管理または DM通報へ
  - 🟡 黄バナー: 未処理通報 5件以上
  - 🔴 赤バナー: 未処理通報 10件以上
- **7日間グラフ** で投稿数・登録数・通報数の傾向を確認
- 最新通報テーブルの「削除+対処」で投稿を即削除して通報を actioned にできる

---

### 🚨 通報管理（掲示板投稿）

**通常フロー**
1. フィルタを「未処理」に設定して内容確認
2. 問題なし → 「確認済」ボタン（`reviewed`）
3. 削除が必要 → 「投稿削除」ボタン（`is_deleted = true` + 通報 `actioned`）
4. 悪質ユーザー → 「警告」または「BAN」

**一括操作（複数件まとめて処理）**
1. 各行の左端チェックボックスで対象を選択
2. ヘッダーのチェックボックスで全選択も可
3. 上部に一括操作バーが出現 → 「一括確認済み」「一括却下」「一括警告」「一括BAN」

---

### 💬 DM通報（画像メッセージ対応）

**画像メッセージの確認**
- `message_type = image` の行にサムネイルが表示される
- 「🖼️ 画像確認」→ フルサイズビューアーが開く
- ビューアー内「🗑️ Storageから削除」→ Supabase Storage から画像を完全削除
- 「🗑️ 画像削除」ボタン（テーブル内）からも直接削除可能

**削除時の動作**
- Storage の `dms/` バケットからファイルを削除
- `chat_reports.status` を `actioned` に更新
- `action_logs` に `delete_dm` ログを記録

---

### 👥 ユーザー管理

| 操作 | 動作 |
|------|------|
| 警告 | `ban_status = warned`、`violation_count + 1`、ログ記録 |
| BAN | `ban_status = banned`、次回アクセス時に強制サインアウト |
| BAN解除 | `ban_status = active` に戻す |
| 詳細 | 投稿数・タイプ・マッチング状態・違反回数を確認。マッチング解放/取消もここから |

> ユーザーを BANすると、そのユーザーが次にページ遷移した瞬間に強制サインアウト＋メッセージ表示される（リアルタイムではない）

---

### 📝 投稿管理（キーワード検索）

- 「内容・ユーザーIDで検索」欄にキーワードを入力すると、`posts.content` を部分一致検索
- 「表示中」タブ: アクティブな投稿のみ表示（通常はこちら）
- 「削除済み」タブ: `is_deleted = true` の投稿。「復元」ボタンで戻せる
- 掲示板投稿に添付された画像はサムネイルで表示される

---

### 📢 お知らせ投稿

1. 「＋ 新規お知らせ」ボタン
2. タイトル（100文字）・内容（2000文字）を入力
3. 「ピン留め」チェックで最上部に固定表示
4. アプリ内の `announcements` テーブルに保存される（現時点ではUIへの表示実装は別途確認）

---

### 📋 アクションログ

- 管理者が行った BAN / 警告 / 投稿削除 / DM削除 の全操作が記録される
- フィルタで操作種別を絞り込み可能
- 捜査機関等への開示要請時に使用

| 操作を見返す | アクションログ |
| お知らせを出す | お知らせ → 新規作成 |
